Workforce Optimisation & Salesforce Transformation

Improving daily care-worker scheduling, strengthening source-data quality and preparing operational capabilities for migration into Salesforce.

03

Building the Rostering Data Flow

Australian Unity engaged Biarri to support workforce rostering. I validated the business requirements and designed the daily data-supply process from the legacy SQL Server environment.

I replaced the existing master extraction procedure with a customised stored procedure returning only the required data. Python generated the CSV outputs, while SQL Server Agent scheduled the process for 6:00 am each morning.

Legacy SQL Server→Custom Stored Procedure→Python CSV Processing→SQL Agent→SFTP→Biarri

I also coordinated secure SFTP setup so input files could be sent to Biarri and processed results returned.

04

A Turning Point

When scheduling outcomes were inconsistent, I traced the issue through the data flow and found that the main problem was not the optimisation engine—it was poor source-data quality.

Missing dataMandatory attributes were incomplete.
Incorrect valuesSource records contained invalid information.
Wrong mappingsSome fields were aligned incorrectly.
Onboarding gapsNew records were not always complete.
What I changed

Built Tableau dashboards showing data issues directly to operations teams.

Process improvement

Recommended onboarding procedure updates for new customers and care workers.

Result

Improved data supplied to the rostering solution and reduced recurring issues.

05

Operational Automation

Biarri returned compressed output files. I created Python and SQL Agent processes to extract and reprocess them, add protected health information deliberately excluded from vendor input, refresh dashboard datasets and place final files into a controlled location.

I then worked with the WebMethods team to automate delivery into the correct SharePoint location for each branch.
